# 🌐 XrayInChrome

**XrayInChrome** — это мощный инструмент, который интегрирует управление ядром Xray непосредственно в боковую панель браузера. Благодаря механизму Chrome Native Messaging, он обеспечивает бесшовную связь между пользовательским интерфейсом браузера и системными сетевыми процессами, позволяя пользователям эффективно управлять сетевыми прокси-серверами без необходимости работы со сложными командными строками или громоздкими конфигурационными файлами.

## ✨ Основные возможности

- **🚀 Минималистичный центр управления**: управление запуском, остановкой и мониторингом состояния ядра Xray в реальном времени через боковую панель Chrome.
- **📦 Интеллектуальное управление подписками**: поддержка импорта ссылок на узлы одним кликом, автоматический парсинг и синхронизация с локальным состоянием без ручного редактирования JSON.
- **📜 Поток логов в реальном времени**: просмотр логов работы Xray непосредственно в интерфейсе браузера для быстрого поиска и устранения проблем с соединением.
- **⚙️ Автоматизированное развертывание**: встроенный Native Host на языке Go с установкой записей реестра одним кликом для безопасной связи между расширением и системными процессами.
- **🛡️ Архитектура разделения привилегий**: использование архитектуры «фронтенд с низкими привилегиями + бэкенд с высокими привилегиями» для обеспечения безопасности среды браузера.

## 📐 Системная архитектура

XrayInChrome использует типичную архитектуру **разделения привилегий (Privilege Separation)**, что обеспечивает как удобство эксплуатации, так и безопасность системы:

```text
[ Браузер Chrome ] 
       │
       ▼
[ Sidepanel UI ]  <───>  [ Background Script ]
                                 │
                                 │ (Chrome Native Messaging API)
                                 ▼
                         [ Go Native Host ]  <───>  [ Реестр Windows ]
                                 │
                                 │ (Управление процессами)
                                 ▼
                         [ Процесс Xray Core ]  <───>  [ Сетевой трафик ]
```

- **Frontend (Extension)**: отвечает за пользовательский интерфейс, парсинг подписок и оркестрацию конфигураций.
- **Native Host (Go)**: отвечает за управление жизненным циклом процесса Xray, физическую генерацию конфигурационных файлов и системные вызовы.
- **Xray Core**: исполняемый файл, реализующий логику сетевого проксирования.

## 🚀 Быстрый старт

### 👤 Руководство по установке для пользователя

Следуйте этим шагам для настройки XrayInChrome:

1. **Получение и запуск настройки**: Посетите [GitHub Репозиторий](https://github.com/warriorrage/XrayInChrome/tree/main/native-host), чтобы скачать и запустить мастер настройки `xray-bridge.exe`.
2. **Подтверждение намерения**: Выберите **'Yes'** в появившемся диалоговом окне, чтобы начать установку или обновление моста.
3. **Привязка ядра**: Выберите файл ядра `xray.exe` на вашем компьютере и нажмите OK. Это регистрирует мост в реестре Windows, предоставляя Chrome разрешение на вызов Xray.
4. **Проверка соединения**: Перезапустите браузер и нажмите кнопку расширения **'Xray in Chrome'**, чтобы открыть боковую панель. Мост запустится автоматически; убедитесь, что статус `xray-bridge.exe` стал **'Работает'**.
5. **Настройка узлов**: Импортируйте прокси-узлы в разделе **'Сервер'** и выберите целевой узел (отмечен ✅) в качестве активной конфигурации.
6. **Запуск ядра**: Включите переключатель в правом верхнем углу **'Главной'**. Это официально запустит процесс ядра `xray.exe`. Когда статус станет **'Работает'** и в логах не будет ошибок, запуск завершен успешно.
7. **Маршрутизация трафика**: В разделе **'Другое'** (Системные настройки) отметьте **'Использовать прокси Xray'** и нажмите **'Применить'**. Трафик браузера теперь будет перенаправляться через Xray.

## 📂 Структура проекта

```text
XrayInChrome/
├── extension/              # Исходный код расширения Chrome
│   ├── manifest.json       # Манифест расширения (V3)
│   ├── background.js       # Фоновый сервис, поддерживает связь с Native Host
│   ├── sidepanel.js        # Основная логика боковой панели
│   ├── state.js            # Централизованное управление состоянием
│   ├── ui-core.js          # Базовые UI компоненты
│   ├── log-manager.js      # Обработка потока логов
│   └── utils.js            # Общие вспомогательные функции
└── native-host/            # Исходный код бэкенда на Go
    ├── main.go             # Точка входа, обработка протокола Native Messaging
    └── pkg/                # Основные пакеты
        ├── messaging/      # Сериализация сообщений и протокол связи
        ├── xray/           # Управление жизненным циклом процесса Xray
        ├── platform/       # Платформенно-зависимые реализации (Реестр Windows и т.д.)
        ├── config/         # Генерация и парсинг конфигураций Xray
        └── installer/      # Логика установки и удаления
```

## ⚠️ Примечание по безопасности

Данный проект работает через официально поддерживаемый Chrome механизм `Native Messaging`. Это означает:
- **Контролируемая связь**: расширение не может произвольно получать доступ к вашей файловой системе; все системные операции проходят строгую проверку в `native-host`.
- **Явное разрешение**: установка бэкенда требует прав администратора (изменение реестра), что гарантирует, что установка осознанна и санкционирована пользователем.
- **Минимальные привилегии**: расширение запрашивает только необходимые API-разрешения и не запускает никаких сторонних удаленных скриптов.
